## 3. 核心代码结构

### 3.1 应用入口 - EntryAbility.ets

**文件位置**: `entry/src/main/ets/entryability/EntryAbility.ets`

**核心职责**:
- 应用生命周期管理
- 窗口初始化配置
- 状态栏高度获取

**关键代码解析**:

```typescript
export default class EntryAbility extends UIAbility {
  // 应用创建时初始化
  onCreate(want: Want, launchParam: AbilityConstant.LaunchParam): void {
    // 设置颜色模式
    this.context.getApplicationContext().setColorMode(
      ConfigurationConstant.ColorMode.COLOR_MODE_NOT_SET
    );
  }

  // 窗口阶段创建
  onWindowStageCreate(windowStage: window.WindowStage): void {
    // 加载主页面
    windowStage.loadContent('pages/Index', (err) => {
      // 设置全屏显示
      windowClass.setWindowLayoutFullScreen(true);
      
      // 获取状态栏高度并存储到 AppStorage
      let avoidArea = windowClass.getWindowAvoidArea(window.AvoidAreaType.TYPE_SYSTEM);
      let topRectHeight = avoidArea.topRect.height;
      AppStorage.setOrCreate('topRectHeight', topRectHeight);
    });
  }
}
```

**技术要点**:
1. 使用 `UIAbility` 作为应用基类
2. 通过 `windowStage.loadContent()` 加载首页
3. 使用 `AppStorage` 全局存储状态栏高度，供页面使用

## 4. 数据模型设计

### 4.1 数据模型定义

**文件位置**: `entry/src/main/ets/common/CommonConstants.ets`

#### 4.1.1 消息对象 (messageObj)

```typescript
export class messageObj {
  sender: string = '';      // 发送者标识: 'others' | 'myself'
  msg: ResourceStr = ''     // 消息内容
}
```

**字段说明**:
- `sender`: 区分消息发送方
  - `'others'`: 对方发送的消息（显示在左侧）
  - `'myself'`: 自己发送的消息（显示在右侧）
- `msg`: 消息文本内容，支持字符串或资源引用

#### 4.1.2 联系人项 (Item)

```typescript
@Observed
export class Item {
  id: number = 0;                        // 唯一标识
  image: Resource = $r('app.media.Public_avatar');  // 头像资源
  name: string = '';                     // 联系人名称
  message: Array<messageObj> = [];       // 消息列表
  isNewMessage: boolean = false;         // 是否有新消息
  time: ResourceStr = '';                // 消息时间
  @Track isTop: boolean = false;         // 是否置顶
}
```

**装饰器说明**:
- `@Observed`: 使类具有观察者能力，支持深度观察属性变化
- `@Track`: 标记属性需要单独追踪，优化渲染性能

**字段详解**:
| 字段 | 类型 | 说明 |
|------|------|------|
| id | number | 联系人唯一标识，用于列表渲染和排序 |
| image | Resource | 头像图片资源引用 |
| name | string | 联系人显示名称 |
| message | Array<messageObj> | 与该联系人的所有聊天消息 |
| isNewMessage | boolean | 是否有未读新消息，控制红点显示 |
| time | ResourceStr | 最后一条消息的时间 |
| isTop | boolean | 是否置顶，影响列表排序和背景色 |

## 5. 页面组件详解

### 5.1 联系人列表页 - Index.ets

**文件位置**: `entry/src/main/ets/pages/Index.ets`

#### 5.1.1 组件结构

```typescript
@Entry
@Component
struct Index {
  @State isFlag: boolean = false;                    // 控制回到顶部按钮显示
  @Provide('NavPathStack') pageStack: NavPathStack = new NavPathStack();  // 导航栈
  @State sortedList: Item[] = [];                    // 排序后的联系人列表
  @State dragItem: Item = {...};                     // 当前拖拽项
  private scroller: Scroller = new Scroller();       // 列表滚动控制器
  @StorageProp('topRectHeight') topRectHeight: number = 0;  // 状态栏高度
}
```

**装饰器说明**:
- `@Entry`: 标记为页面入口组件
- `@Component`: 定义自定义组件
- `@State`: 状态变量，变化时触发 UI 刷新
- `@Provide`: 提供数据给子组件消费
- `@StorageProp`: 从 AppStorage 单向同步数据

#### 5.1.2 核心方法

##### (1) 列表排序 - sortList()

```typescript
sortList() {
  this.sortedList = CommonConstants.MESSAGE_LIST.sort((a, b) => {
    if (a.isTop && !b.isTop) {
      return -1;  // a 置顶，排在前面
    } else if (!a.isTop && b.isTop) {
      return 1;   // b 置顶，排在前面
    }
    return a.id - b.id;  // 都置顶或都不置顶，按 id 排序
  })
}
```

**排序逻辑**:
1. 置顶项始终排在列表顶部
2. 非置顶项按 id 升序排列
3. 置顶项之间也按 id 排序

##### (2) 切换置顶状态 - toggleTop()

```typescript
toggleTop(item: Item) {
  item.isTop = !item.isTop;  // 切换置顶状态
  this.sortList();           // 重新排序
}
```

#### 5.1.3 UI 构建

##### (3) 拖拽排序

```typescript
// 开始拖拽
.onItemDragStart((event: ItemDragInfo, itemIndex: number) => {
  this.dragItem = this.sortedList[itemIndex];
  return this.dragFloatView(this.sortedList[itemIndex]);  // 返回拖拽浮层
})

// 拖拽移动
.onItemDragMove((event: ItemDragInfo, itemIndex: number, insertIndex: number) => {
  this.getUIContext().animateTo({ duration: 200, curve: Curve.Linear }, () => {
    let deleteIndex = this.sortedList.indexOf(this.dragItem);
    this.sortedList.splice(deleteIndex, 1);        // 移除原位置
    this.sortedList.splice(insertIndex, 0, this.dragItem);  // 插入新位置
  })
})

// 拖拽结束
.onItemDrop(() => {
  this.dragItem = {...};  // 重置拖拽项
})
```

```typescript
// 滚动监听
```

### 5.2 聊天详情页 - ChatPage.ets

**文件位置**: `entry/src/main/ets/pages/ChatPage.ets`

#### 5.2.1 组件结构

```typescript
@Component
export struct ChatPage {
  private scroller: Scroller = new Scroller();       // 滚动控制器
  @State messageList: Array<string> = [];            // 消息列表（未使用）
  @State inputMessage: string = '';                  // 输入框内容
  @State isRefreshing: boolean = false;              // 下拉刷新状态
  pathStack: NavPathStack = new NavPathStack();      // 导航栈
  @State itemInfo: Item = {...};                     // 当前联系人信息
  @State itemMessage: messageObj[] = [];             // 所有消息
  @State showItemMessage: messageObj[] = [];         // 当前显示的消息
  @State startIndex: number = 0;                     // 消息起始索引
}
```

#### 5.2.2 核心方法

##### (1) 加载更多消息 - getLastTenElements()

```typescript
getLastTenElements(arr: messageObj[]) {
  this.startIndex = Math.max(0, this.startIndex -= 10);  // 向前移动10条
  
  if (!Array.isArray(arr) || arr.length < 10 || this.startIndex === 0) {
    this.showItemMessage = arr;  // 显示全部消息
    return;
  }
  
  // 提取前10条消息并插入到顶部
  const lastTenElements = arr.slice(this.startIndex, this.startIndex + 10);
  this.showItemMessage.unshift(...lastTenElements);
}
```

**加载逻辑**:
1. 每次下拉加载 10 条历史消息
2. 使用 `unshift()` 将历史消息插入到列表顶部
3. 当 `startIndex === 0` 时，显示全部消息

#### 5.2.3 UI 构建

##### (1) 消息列表

```typescript
Refresh({ refreshing: $$this.isRefreshing }) {
  List({ space: 10, scroller: this.scroller, 
         initialIndex: this.itemInfo.message.length - 1 }) {
    ForEach(this.showItemMessage, (item: messageObj, index: number) => {
      ListItem() {
        if (item.sender === 'others') {
          // 对方消息（左侧）
          Row() {
            Image(this.itemInfo.image).width(36).height(36)
            Text(item.msg)
              .backgroundColor('#F1F3F5')
              .borderRadius(12)
          }
          .justifyContent(FlexAlign.Start)
        } else {
          // 自己消息（右侧）
          Row() {
            Text(item.msg)
              .backgroundColor('#F1F3F5')
              .borderRadius(12)
            Image($r('app.media.Public_avatar')).width(36).height(36)
          }
          .justifyContent(FlexAlign.End)
        }
      }
    })
  }
  .onAppear(() => {
    this.scroller.scrollEdge(Edge.Bottom);  // 初始滚动到底部
  })
}
.onRefreshing(() => {
  setTimeout(() => {
    this.getLastTenElements(this.itemMessage);  // 加载更多
    this.isRefreshing = false;
  }, 1500)
})
```

**关键特性**:
- `initialIndex`: 初始显示最后一项（最新消息）
- `Refresh` 组件实现下拉刷新
- 根据发送者显示不同布局

##### (2) 输入框和发送

```typescript
TextInput({ placeholder: 'input your word...', text: this.inputMessage })
  .onChange((value) => {
    this.inputMessage = value;
  })

Image(this.inputMessage === '' ? $r('app.media.send_off') : $r('app.media.send_on'))
```

### 9.4 性能优化建议

1. **使用 @Track 优化渲染**
   - 对频繁变化的属性使用 `@Track` 装饰器
   - 避免整个对象重新渲染

2. **合理使用 ForEach**
   - 提供唯一的 key 生成函数
   - 示例: `(item: Item) => item.id.toString() + item.isTop`

3. **控制列表渲染数量**
   - 使用分页加载，避免一次性渲染大量数据
   - 示例: `slice(-10)` 只显示最新 10 条

4. **使用 LazyForEach**
   - 对于大数据量列表，使用 `LazyForEach` 替代 `ForEach`
   - 实现按需加载，减少内存占用

### 9.5 最佳实践

1. **数据不可变性**
   - 使用 `JSON.parse(JSON.stringify())` 深拷贝对象
   - 避免直接修改原始数据

2. **动画过渡**
   - 状态变化时使用 `animateTo` 添加动画
   - 提升用户体验

3. **资源引用**
   - 使用 `$r('app.media.xxx')` 引用资源
   - 支持多语言和多分辨率

4. **生命周期管理**
   - `aboutToAppear`: 组件即将出现时初始化
   - `aboutToDisappear`: 组件即将消失时清理资源

```typescript
Refresh({ refreshing: $$this.isRefreshing })
```

```typescript
List({ initialIndex: this.itemInfo.message.length - 1 })
```

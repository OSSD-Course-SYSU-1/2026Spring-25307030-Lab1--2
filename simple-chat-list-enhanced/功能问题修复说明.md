### 2. ✅ 草稿不能保存

**问题描述**:
- 在输入框输入内容后，切换到其他聊天或返回列表
- 再次进入该聊天时，之前输入的内容丢失

**原因分析**:
- `itemInfo` 是通过深拷贝（`JSON.parse(JSON.stringify())`）创建的
- 修改 `itemInfo.draft` 不会影响原始数据
- 每次进入聊天页面都会重新从原始数据加载

**解决方案**:
使用静态 Map 存储每个联系人的草稿

**修改位置**: ChatPage.ets

#### (1) 添加静态草稿存储

```typescript
// 草稿存储（使用 Map 存储每个联系人的草稿）
private static draftMap: Map<number, string> = new Map();
private currentContactId: number = 0;  // 当前联系人ID
```

#### (2) 修改草稿保存逻辑

**修改前**:
```typescript
onInputChange(value: string) {
  this.inputMessage = value;
  
  // 保存草稿
  this.itemInfo.draft = value;  // ❌ 不会持久化
  
  // ...
}
```

**修改后**:
```typescript
onInputChange(value: string) {
  this.inputMessage = value;
  
  // 保存草稿到静态 Map
  ChatPage.draftMap.set(this.currentContactId, value);  // ✅ 持久化存储
  
  // ...
}
```

#### (3) 修改草稿恢复逻辑

**修改前**:
```typescript
.onReady((content: NavDestinationContext) => {
  this.itemInfo = JSON.parse(JSON.stringify(content.pathInfo.param));
  this.itemMessage = this.itemInfo.message;
  this.startIndex = this.itemMessage.length - 10;
  this.showItemMessage = this.itemMessage.slice(-10);
  
  // 恢复草稿
  if (this.itemInfo.draft) {  // ❌ itemInfo 是新创建的，没有草稿
    this.inputMessage = this.itemInfo.draft;
  }
})
```

**修改后**:
```typescript
.onReady((content: NavDestinationContext) => {
  this.itemInfo = JSON.parse(JSON.stringify(content.pathInfo.param));
  this.currentContactId = this.itemInfo.id;  // ✅ 保存当前联系人ID
  this.itemMessage = this.itemInfo.message;
  this.startIndex = this.itemMessage.length - 10;
  this.showItemMessage = this.itemMessage.slice(-10);
  
  // 恢复草稿
  let savedDraft = ChatPage.draftMap.get(this.currentContactId);  // ✅ 从 Map 获取
  if (savedDraft) {
    this.inputMessage = savedDraft;
  }
})
```

**效果**:
- 输入内容时自动保存草稿
- 切换聊天或返回列表后，草稿仍然保存
- 再次进入该聊天时，自动恢复草稿
- 每个联系人的草稿独立保存

## 技术要点

### 1. 静态变量

**用途**: 在组件实例之间共享数据

```typescript
class MyComponent {
  // 静态变量，所有实例共享
  private static sharedData: Map<number, string> = new Map();
  
  // 实例变量，每个实例独立
  private instanceData: string = '';
}
```

**特点**:
- 所有组件实例共享同一个静态变量
- 适合存储需要跨实例共享的数据（如草稿）

### 2. Map 数据结构

**用途**: 存储键值对

```typescript
// 创建 Map
const draftMap = new Map<number, string>();

// 设置值
draftMap.set(contactId, draftText);

// 获取值
const draft = draftMap.get(contactId);

// 检查是否存在
if (draftMap.has(contactId)) {
  // ...
}

// 删除
draftMap.delete(contactId);

// 清空
draftMap.clear();
```

**优势**:
- 键可以是任意类型（对象、数字等）
- 提供丰富的方法
- 性能优于对象

### 3. 深拷贝问题

**问题**: 深拷贝会创建全新的对象，修改不会影响原对象

```typescript
// 深拷贝
const copy = JSON.parse(JSON.stringify(original));

// 修改拷贝
copy.property = 'new value';

// 原对象不受影响
console.log(original.property);  // 仍然是旧值
```

**解决方案**:
- 使用静态变量存储需要持久化的数据
- 或使用状态管理框架（如 AppStorage）

## 注意事项

### 1. 静态变量的生命周期

- 静态变量在应用运行期间一直存在
- 应用关闭后数据会丢失
- 如需持久化，应使用本地存储

### 2. 草稿存储限制

- 当前实现使用内存存储
- 应用重启后草稿会丢失
- 如需持久化，可以使用：
  - AppStorage
  - PersistentStorage
  - 本地数据库

### 3. 录音功能限制

- 当前为模拟录音
- 实际使用需要：
  - 添加录音权限
  - 使用 @kit.AudioKit
  - 保存录音文件

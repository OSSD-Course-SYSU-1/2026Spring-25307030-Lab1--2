## 最后修复的错误（6个）

### 问题：messageObj | null 不能赋值给 messageObj

**错误信息**:
```
Argument of type 'messageObj | null' is not assignable to parameter of type 'messageObj'.
Type 'null' is not assignable to type 'messageObj'.
```

**原因**: 
- `selectedMessage` 的类型是 `messageObj | null`
- 但函数参数期望的是 `messageObj`（不能为 null）

**解决方案**: 添加空值检查

## TypeScript 空值处理最佳实践

### 1. 可选链操作符 (?.)

```typescript
// 安全访问属性
const sender = this.selectedMessage?.sender;

// 安全调用方法
this.selectedMessage?.method?.();
```

### 2. 空值合并操作符 (??)

```typescript
// 提供默认值
const message = this.selectedMessage ?? defaultMessage;
```

### 3. 类型守卫

```typescript
// 检查是否为 null
if (this.selectedMessage) {
  // 这里 TypeScript 知道 selectedMessage 不是 null
  this.recallMessage(this.selectedMessage);
}
```

### 4. 非空断言 (!)

```typescript
// 仅在确定不为 null 时使用
this.recallMessage(this.selectedMessage!);  // ⚠️ 危险，不推荐
```

## 技术总结

### 关键点

1. **类型安全**: TypeScript 严格模式要求处理所有可能的值
2. **空值检查**: 在使用可能为 null 的值前必须检查
3. **可选类型**: 使用 `Type | null` 明确表示可能为空
4. **防御性编程**: 始终假设值可能为空

### 学到的经验

1. ✅ 定义类型时考虑可能的空值
2. ✅ 使用前检查是否为空
3. ✅ 提供合理的默认值
4. ✅ 使用 TypeScript 的类型系统帮助发现错误
